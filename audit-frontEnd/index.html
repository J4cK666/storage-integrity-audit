算法流程

User

用户

用户负责系统初始化、文件处理、关键词索引生成以及审计查询准备，是整个关键词审计链路的入口。

Init 初始化系统参数，例如密钥等加密参数。
Setup 将文件拆分、加密并准备上传到云端。
IndexGenerate 构建安全索引，为关键词搜索与审计定位提供依据。
AuthenticatorGenerate 为每个加密数据块生成验证器。
TrapdoorGenerate 根据查询关键词生成用于审计的搜索陷门。
Cloud

云端

云端持有被加密存储的文件与相关辅助信息，在接收到 TPA 挑战后快速生成可验证的审计证明。

ProofGenerate 根据 TPA 发出的挑战和关键词查询范围生成审计证明。
TPA

TPA

TPA 作为第三方审计者，根据用户提供的搜索陷门生成挑战，并对云端返回的证明执行完整性验证。

ChallengeGenerate 根据搜索陷门生成审计挑战，将审计聚焦到包含目标关键词的文件集。
ProofVerify 验证云端生成的审计证明，检查目标文件的完整性。
